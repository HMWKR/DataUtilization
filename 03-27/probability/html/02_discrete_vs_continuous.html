왜 이산형과 연속형을 구분하는가?

확률변수(Random Variable)는 실험 결과를 수치로 대응시킨 것입니다. 이 수치가 "셀 수 있는" 값인지, "측정하는" 값인지에 따라 확률을 다루는 방법이 완전히 달라집니다.

왜 이 구분이 중요한가요?

이산형은 확률질량함수(PMF)로, 연속형은 확률밀도함수(PDF)로 다룹니다. 둘을 섞으면 확률 계산이 완전히 틀어집니다.

예: 주사위에서 $$P(X=3)=1/6$$ 은 의미가 있지만, 키에서 $$P(X=170.000...)=0$$ 입니다. 연속형에서는 구간 확률만 의미가 있습니다.

1. 이산형 변수란? - 셀 수 있는 것들

핵심 정의

확률변수 $$X$$가 취할 수 있는 값을 하나, 둘, 셋... 으로 셀 수 있으면 이산형입니다.

양의 정수(positive integer)로 세어나갈 수 있으면 이산, 그렇지 않으면 연속이다.

대표적인 예시

실험확률변수 X가능한 값셀 수 있나?
동전 던지기앞면 수{0, 1}O (2개)
주사위 1개나온 눈{1, 2, 3, 4, 5, 6}O (6개)
카드 뽑기카드 종류{스페이드, 하트, 다이아, 클럽}O (4개)
불량품 검사불량 개수{0, 1, 2, ..., n}O (유한)
전화 횟수/일수신 건수{0, 1, 2, 3, ...}O (무한이지만 가산)

확률질량함수 (PMF)

이산형 변수에서는 각 값에 확률을 직접 부여할 수 있습니다.

$$P(X=k) \ge 0 \quad \text{이고} \quad \sum_{\text{모든 } k} P(X=k) = 1$$

주사위라면: $$P(X=1) = P(X=2) = \cdots = P(X=6) = \frac{1}{6}$$

이산형의 핵심 특징 3가지
  • 셈(counting)이 가능하다 - 값 사이에 "빈틈"이 있다
  • 각 값에 확률을 부여할 수 있다 - $$P(X=3)=1/6$$
  • 확률의 합이 1 - 모든 가능한 값의 확률을 더하면 정확히 1

2. 연속형 변수란? - 측정하는 것들

핵심 정의

확률변수 $$X$$가 실수 구간의 모든 값을 취할 수 있으면 연속형입니다. 값 사이에 빈틈이 없고, 어떤 두 값 사이에도 무한히 많은 값이 존재합니다.

실수형으로 연속되어 정확하게 관측하지 못하면 연속(Continuous)이다.

대표적인 예시

측정 대상변수왜 연속형?
사람의 키170.3cm? 170.31cm?소수점 아래 끝없이 세분화 가능
몸무게65.2kg? 65.21kg?측정 정밀도에 따라 무한히 세분화
시간1시 10분 00.000...초시간은 연속적으로 흐른다
나이25세? 25.123...세?나이도 실수로 연속
온도36.5도? 36.51도?무한한 정밀도

X=1 vs X=1.0 의 차이

이산형에서 $$X=1$$은 명확합니다. 하지만 연속형에서 $$X=1.0$$이란?

이산형: X = 1      (정확히 1, 다른 값과 구분됨)
연속형: X = 1.0    (1.0000000000...?)
        X = 1.00000000001?  ← 이것도 "거의" 1인데?
        X = 0.99999999999?  ← 이것은?

연속형에서는 "정확히 1.0"이라는 점 하나의 확률이 0입니다. 이것이 연속형의 핵심 성질입니다.

확률밀도함수 (PDF)

$$P(a \le X \le b) = \int_a^b f(x)\,dx$$

연속형에서는 구간의 넓이(적분)로만 확률을 구할 수 있습니다.

3. 왜 정확한 값을 알 수 없는가?

당신의 정확한 키를 아시나요?

병원에서 "170cm"라고 측정받았다고 합시다. 하지만:

  • 정밀한 기계: 170.3cm
  • 더 정밀한 기계: 170.312cm
  • 원자 수준: 170.312847291038...cm

측정 도구의 정밀도가 높아질수록 숫자는 끝없이 이어집니다. "정확한" 키는 무한 소수이고, 우리는 절대 그 모든 자릿수를 알 수 없습니다.

컴퓨터도 마찬가지: 0.1 + 0.2 문제

Python으로 직접 확인해봅시다:

print(0.1 + 0.2)
0.30000000000000004

소수점 아래 16번째 자리에서 4가 튀어나옵니다! 이것은 Python의 버그가 아닙니다. 64비트(double precision)로 실수를 표현하는 모든 컴퓨터에서 발생하는 현상입니다.

왜 이런 일이 생기는가?

10진수 0.1을 2진수로 변환

$$0.1_{10} = 0.0001100110011001100..._2$$

2진수에서 0.1은 무한 순환소수입니다!

64비트에 잘라서 저장

IEEE 754 표준: 부호(1비트) + 지수(11비트) + 가수(52비트) = 64비트

52비트로 무한소수를 잘라야 하므로 반올림 오차 발생

오차가 누적

0.1의 근사값 + 0.2의 근사값 = 0.3의 근사값이 아닌 0.30000000000000004

교수님이 전달하시는 핵심 메시지

연속형 변수의 "정확한 값"이란 이론적으로만 존재합니다. 인간도, 컴퓨터도 유한한 정밀도를 갖기 때문에 실수를 "정확히" 표현하거나 관측할 수 없습니다.

이것이 바로 연속형 변수에서 "한 점의 확률이 0"인 물리적 직관입니다.

4. 연속형에서 P(X=정확히 k) = 0인 이유

직관적 이해: 버스 문제

당신이 "정확히 1시 10분 00.000...초"에 버스를 탈 확률은 얼마일까요?

"1시10분에 버스를 탈 확률은?" - 정확히 그 순간의 확률은 0입니다.

왜일까요? 시간은 연속적으로 흐릅니다:

  • 1시 10분 00.000000001초도 "1시 10분"이 아닙니다
  • 1시 09분 59.999999999초도 "1시 10분"이 아닙니다
  • 정확히 1:10:00.000...인 "점"은 시간 축 위의 무한히 작은 점 하나
  • 점 하나의 "길이(폭)"는 0이므로, 확률(면적)도 0

수학적 이해

$$P(X = k) = \int_k^k f(x)\,dx = 0$$

적분 구간의 폭이 0이면 면적도 0입니다. 아무리 $$f(k)$$가 크더라도!

하지만 "불가능"은 아닙니다

$$P(X=k)=0$$이라고 해서 $$X=k$$가 일어나지 않는다는 뜻이 아닙니다. 확률 0인 사건도 일어날 수 있습니다. 이것이 연속 확률의 독특한 점입니다.

구간 확률은 의미가 있습니다

"1시 10분 전후 30초 사이에 버스를 탈 확률"은 의미 있는 양수입니다.

$$P(1:09:30 \le X \le 1:10:30) = \int_{1:09:30}^{1:10:30} f(x)\,dx > 0$$

연속형에서는 항상 구간으로 물어야 의미 있는 확률을 얻습니다.

왜 이것이 P(X=k)=0과 연결되는가?

한 점 $$k$$를 찍어도, 그 점의 "폭"은 0입니다. 전체 구간의 길이에서 점 하나가 차지하는 비율은:

$$\frac{0}{\text{구간 길이}} = 0$$

확대할수록 더 많은 점이 보이지만, 각 점의 폭은 여전히 0입니다. 이것이 연속형 변수에서 한 점의 확률이 0인 기하학적 이유입니다.

5. ML에서의 의미 - 분류 vs 회귀

이산형과 연속형의 구분은 머신러닝에서 가장 기본적인 문제 분류와 직결됩니다.

분류 (Classification)

예측 대상이 이산형일 때

  • 스팸 메일? 정상? (0 또는 1)
  • 고양이? 강아지? 새? (3개 클래스)
  • 신용 등급: Good / Bad
  • 질병 진단: 양성 / 음성

출력: 이산적 레이블(label)

손실함수: Cross-entropy

평가지표: 정확도, F1, AUC

회귀 (Regression)

예측 대상이 연속형일 때

  • 집 가격은? (연속적 금액)
  • 내일 기온은? (실수)
  • 매출 예측 (연속적 금액)
  • 주가 예측 (실수 값)

출력: 연속적 실수값

손실함수: MSE, MAE

평가지표: RMSE, R-squared

경계가 모호한 경우는?

시험 점수는 이산? 연속? - 0~100점의 정수라면 이산형이지만, 실무에서 연속형 근사하기도 합니다.

나이는 보통 정수(25세)로 기록하지만 본질적으로 연속형입니다.

중요한 것은 문제를 어떻게 모델링하느냐입니다. 같은 변수도 상황에 따라 이산/연속으로 다르게 취급할 수 있습니다.
