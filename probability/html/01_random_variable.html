1. 확률변수란?

확률과 통계를 배우기 전에, 먼저 핵심 용어들을 정확히 이해하고 넘어가겠습니다. 이 용어들은 앞으로 모든 확률 이론의 기초가 됩니다.

📚 핵심 용어 정리

실험 (Experiment)
결과를 관찰하기 위해 수행하는 행위입니다. 예: 주사위 던지기, 동전 던지기, 제품 검사 등. 같은 조건에서 반복해도 결과가 달라질 수 있는 행위를 확률적 실험(Random Experiment)이라 합니다.
표본공간 (Sample Space, $S$)
실험에서 나올 수 있는 모든 결과의 집합입니다. 주사위 1개를 던지면 $S = \{1, 2, 3, 4, 5, 6\}$이고, 동전 2번 던지면 $S = \{HH, HT, TH, TT\}$입니다. 가능한 결과를 빠짐없이 모아놓은 것이 핵심입니다.
사건 (Event)
표본공간의 부분집합입니다. "주사위에서 짝수가 나오는 사건" = $\{2, 4, 6\}$처럼, 우리가 관심있는 결과들을 모아놓은 것입니다.
확률 (Probability)
어떤 사건이 일어날 가능성을 0과 1 사이의 숫자로 나타낸 것입니다. 0이면 절대 일어나지 않고, 1이면 반드시 일어납니다. 공정한 주사위에서 $P(\text{짝수}) = 3/6 = 0.5$입니다.
확률변수 (Random Variable, $X$)
표본공간의 각 원소(결과)를 실수(숫자)로 대응시키는 함수입니다. 쉽게 말하면, 실험 결과에 숫자 라벨을 붙이는 규칙입니다. 예를 들어 동전 2번 던지기에서 "앞면의 개수"를 확률변수 $X$라 하면 $X(HH)=2$, $X(HT)=1$, $X(TH)=1$, $X(TT)=0$이 됩니다.
핵심 포인트: 확률변수는 "변수"라는 이름이 붙어있지만, 사실은 함수(function)입니다. 표본공간(입력) → 실수(출력)로 변환하는 함수라는 점을 꼭 기억하세요.
ML/AI와의 연결: 왜 확률변수가 중요한가?

머신러닝에서 확률변수 개념은 다음과 같이 활용됩니다:

  • 특성(Feature): 데이터의 각 열(column)은 확률변수로 모델링됩니다. 키, 몸무게, 나이 등이 모두 확률변수입니다.
  • 확률 분포: 확률변수가 어떤 값들을 어떤 빈도로 가지는지를 확률 분포(Probability Distribution)라 하며, 이것이 통계적 모델링의 핵심입니다.
  • 베이즈 정리: 사후확률 $P(Y|X)$를 계산할 때, $X$와 $Y$ 모두 확률변수입니다.
  • 손실함수: 모델의 예측 오차도 확률변수이며, 기대값(평균)을 최소화하는 것이 학습 목표입니다.

2. 문제로 시작하기

확률 이론을 본격적으로 배우기 전에, 직관적인 주사위 문제 3개를 풀어보겠습니다. 이 문제들을 통해 확률의 기본 원리를 체감할 수 있습니다.

문제 1: 주사위에서 5가 나올 확률

문제: 1개의 주사위를 던져서 5가 나올 확률은?

$$P(X=5) = \frac{\text{5가 나오는 경우의 수}}{\text{전체 경우의 수}} = \frac{1}{6} \approx 0.1667$$

풀이: 공정한 주사위의 표본공간은 $S = \{1, 2, 3, 4, 5, 6\}$으로 총 6개의 결과가 있습니다. 5가 나오는 경우는 딱 1가지이므로, 확률은 $\frac{1}{6}$입니다.

문제 2: 5 이하가 나올 확률

문제: 1개의 주사위를 던져서 5 이하가 될 확률은?

$$P(X \leq 5) = \frac{|\{1, 2, 3, 4, 5\}|}{6} = \frac{5}{6} \approx 0.8333$$

풀이: 5 이하인 숫자는 $\{1, 2, 3, 4, 5\}$로 5개입니다. 전체 6개 중 5개이므로 확률은 $\frac{5}{6}$입니다. 다르게 생각하면, "6이 나오지 않을 확률"과 같으므로 $1 - \frac{1}{6} = \frac{5}{6}$입니다.

문제 3: 두 주사위의 합이 8일 확률

문제: 2개의 주사위를 던지기 게임에서 2개 주사위의 합이 8일 확률은?

$$P(X+Y=8) = \frac{5}{36} \approx 0.1389$$

풀이: 2개의 주사위를 던지면 전체 경우의 수는 $6 \times 6 = 36$가지입니다. 합이 8이 되는 조합은:

  • (2, 6), (3, 5), (4, 4), (5, 3), (6, 2) → 5가지

따라서 확률은 $\frac{5}{36}$입니다.

표본공간 빌더 (Sample Space Builder)

아래 표에서 선택한 합에 해당하는 칸이 강조 표시됩니다. 드롭다운으로 원하는 합을 선택하여 해당 경우를 확인해보세요.

주사위 시뮬레이터 (Dice Simulator)

주사위를 클릭하거나 버튼을 눌러 실험해보세요. 횟수가 늘어날수록 이론적 확률에 수렴하는 것을 관찰할 수 있습니다 (큰 수의 법칙, Law of Large Numbers).

던진 횟수: 0
ML/AI와의 연결: 큰 수의 법칙과 데이터 수집

위 시뮬레이터에서 횟수를 늘리면 관측 빈도가 이론적 확률에 수렴하는 것을 볼 수 있습니다. 이것이 큰 수의 법칙(Law of Large Numbers)입니다.

  • 머신러닝에서 데이터가 많을수록 모델이 정확해지는 이유가 바로 이 법칙 때문입니다.
  • 학습 데이터의 통계적 특성이 모집단(전체 데이터)의 특성에 수렴하기 때문입니다.
  • SGD(확률적 경사하강법)에서 미니배치 크기가 클수록 기울기 추정이 안정적인 것도 같은 원리입니다.

3. 파스칼과 페르마의 도전

📜 역사적 배경: 확률론의 탄생

확률론은 17세기 도박 문제에서 시작되었습니다. 프랑스의 도박사 드 메레(Chevalier de Mere)가 수학자 블레즈 파스칼(Blaise Pascal, 1623~1662)에게 편지를 보내면서 확률론이 본격적으로 발전하기 시작했습니다.

문제: 1억의 상금 배분

A와 B 두 사람이 게임을 합니다. 규칙은 다음과 같습니다:

  • 각 판에서 이길 확률은 50%로 동일
  • 먼저 6번 이기면 총 상금 1억을 가져감
  • 현재 상황: A가 5승, B가 3승인 시점에서 게임 중단
  • 질문: 상금 1억을 어떻게 배분해야 공정한가?

파스칼 (Pascal, 1623~1662)의 풀이

파스칼은 "남은 게임에서 A가 이길 확률"을 계산했습니다:

  • A는 1번만 더 이기면 승리, B는 3번 더 이겨야 승리
  • A가 다음 판에서 바로 이길 확률: $\frac{1}{2}$
  • A가 그 다음 판에서 이길 확률: $\frac{1}{2} \times \frac{1}{2} = \frac{1}{4}$
  • A가 그 다다음 판에서 이길 확률: $\frac{1}{2} \times \frac{1}{2} \times \frac{1}{2} = \frac{1}{8}$
$$P(A\text{ 승리}) = \frac{1}{2} + \frac{1}{4} + \frac{1}{8} = \frac{7}{8} = 87.5\%$$

따라서 A에게 $\frac{7}{8} \times 1\text{억} = 8,750\text{만원}$, B에게 $\frac{1}{8} \times 1\text{억} = 1,250\text{만원}$을 배분하는 것이 공정합니다.

페르마 (Fermat, 1607~1665)의 풀이

페르마는 모든 가능한 경우의 수를 나열하는 방법을 사용했습니다. 최대 3판 더 진행한다고 가정하면:

7번째 판8번째 판9번째 판승자
A 승--A
B 승A 승-A
B 승B 승A 승A
B 승B 승B 승B

총 8가지 경우 중 (각 판이 $\frac{1}{2}$ 확률이므로 $2^3 = 8$가지), A가 이기는 경우가 7가지, B가 이기는 경우가 1가지입니다.

$$P(A) = \frac{7}{8},\quad P(B) = \frac{1}{8}$$

두 수학자가 서로 다른 방법으로 접근했지만 같은 답에 도달했습니다. 이것이 바로 확률론의 아름다움입니다.

ML/AI와의 연결: 확률적 사고와 의사결정

파스칼-페르마 문제의 핵심 아이디어는 현대 AI에도 깊이 녹아있습니다:

  • 기대값(Expected Value): 상금 배분은 "기대 보상"을 계산한 것이며, 강화학습에서 가치 함수(Value Function)와 같은 개념입니다.
  • 몬테카를로 시뮬레이션: 페르마처럼 모든 경우를 나열하는 대신, 실제로 수천 번 시뮬레이션하여 확률을 추정하는 방법이 AI에서 광범위하게 사용됩니다.
  • A/B 테스트: "어떤 모델이 더 좋은가?"를 판단할 때도 확률적 사고가 필수입니다.

4. 확률변수의 정의

확률론(Probability Theory)은 사건의 빈도를 분석하는 학문입니다. 그리고 그 핵심 도구가 바로 확률변수입니다.

📚 확률변수의 엄밀한 정의

확률변수 (Random Variable)
표본공간 $S$의 각 원소(실험 결과)를 실수(real number)에 대응시키는 함수입니다.
수학적 표기: $X: S \rightarrow \mathbb{R}$
쉽게 말하면, "실험 결과에 숫자를 붙이는 규칙"입니다.

예시: 동전 2번 던지기

실험: 동전을 2번 던진다

  • 표본공간: $S = \{HH, HT, TH, TT\}$
  • 확률변수 $X$ = "앞면(Head)의 개수"

이때 확률변수 $X$는 각 결과를 다음과 같이 숫자로 매핑합니다:

실험 결과 (표본공간의 원소)확률변수 $X$의 값의미
TT (뒷면, 뒷면)0앞면 0개
HT (앞면, 뒷면)1앞면 1개
TH (뒷면, 앞면)1앞면 1개
HH (앞면, 앞면)2앞면 2개

확률변수 매핑 시각화 (Event → Number)

왼쪽의 실험 결과(사건)를 마우스로 가리키면, 오른쪽의 숫자(확률변수 값)로 매핑되는 과정이 시각적으로 표시됩니다.

표본공간 S
확률변수 X

확률변수의 확률 계산

확률변수 $X$가 특정 값을 가질 확률은 해당 값에 매핑되는 사건의 확률을 합산하면 됩니다:

$$P(X=0) = P(\{TT\}) = \frac{1}{4} = 0.25$$ $$P(X=1) = P(\{HT, TH\}) = \frac{2}{4} = 0.50$$ $$P(X=2) = P(\{HH\}) = \frac{1}{4} = 0.25$$

이렇게 확률변수의 모든 가능한 값과 그 확률을 나타낸 것을 확률분포(Probability Distribution)라고 합니다. 모든 확률의 합은 반드시 1이 됩니다: $P(X=0) + P(X=1) + P(X=2) = 0.25 + 0.50 + 0.25 = 1$

ML/AI와의 연결: 확률변수와 데이터 특성

머신러닝의 데이터에서 각 특성(feature)은 확률변수로 모델링됩니다:

  • 이산 확률변수(Discrete): 셀 수 있는 값을 가짐. 예: 클릭 여부(0/1), 별점(1~5), 카테고리(A/B/C)
  • 연속 확률변수(Continuous): 연속적인 실수 값. 예: 키, 몸무게, 온도, 주가
  • 분류 모델의 출력 softmax는 각 클래스에 대한 확률분포를 나타냅니다.

동전 던지기 실험 시뮬레이터

동전 2번 던지기를 직접 시뮬레이션해보세요. 실험 횟수가 늘어날수록 상대 빈도가 이론적 확률 (0.25, 0.50, 0.25)에 수렴하는 것을 확인하세요.

실험 횟수: 0
ML/AI와의 연결: 빈도주의 vs 베이즈주의

위 실험처럼 "많이 반복해서 빈도를 세는" 확률 해석을 빈도주의(Frequentist)라고 합니다:

  • 빈도주의: 확률 = 무한 반복 시 사건 발생 비율. 전통적 통계학의 기반.
  • 베이즈주의(Bayesian): 확률 = 믿음의 정도(degree of belief). 사전 지식을 반영할 수 있음.
  • 머신러닝에서 MLE(최대우도추정)는 빈도주의, MAP(최대사후확률추정)는 베이즈주의 접근입니다.
  • 딥러닝에서 드롭아웃(Dropout)은 베이즈 근사로 해석할 수 있습니다.

6. Probability = 1/Surprise

확률을 직관적으로 이해하는 또 다른 방법이 있습니다. 바로 "놀라움의 역수(Inverse of Surprise)"입니다.

$$\text{Probability} = \frac{1}{\text{Surprise}}$$

📚 놀라움(Surprise)이란?

직관적 이해
어떤 사건이 일어났을 때 우리가 느끼는 놀라움의 정도는 그 사건의 확률과 반비례합니다.
  • 공정한 동전에서 앞면: 확률 $\frac{1}{2}$ → 놀라움 낮음 (자주 일어나니까)
  • 주사위에서 6: 확률 $\frac{1}{6}$ → 놀라움 중간
  • 복권 1등 당첨: 확률 $\frac{1}{8,000,000}$ → 놀라움 매우 높음!
정보 이론에서의 정보량 (Self-Information)
클로드 섀넌(Claude Shannon)은 이 아이디어를 수학적으로 발전시켰습니다: $$I(x) = -\log_2 P(x) = \log_2 \frac{1}{P(x)}$$ 확률이 낮을수록 정보량(=놀라움)이 높습니다. 단위는 비트(bit)입니다.

Surprise 미터 (놀라움 측정기)

슬라이더를 움직여서 확률과 놀라움의 관계를 직접 체험해보세요. 확률이 낮아질수록 놀라움(Surprise)이 급격히 증가합니다.

왜 이것이 중요한가?

이 간단한 아이디어 "확률 = 놀라움의 역수"는 놀랍도록 강력한 결과로 이어집니다:

  • 정보 이론(Information Theory): 데이터 압축, 통신의 기초
  • 엔트로피(Entropy): 불확실성의 평균 = 놀라움의 기대값. $H = -\sum P(x)\log_2 P(x)$
  • 크로스 엔트로피(Cross-Entropy): 분류 모델의 손실 함수로 가장 널리 사용
$$H(P, Q) = -\sum_{x} P(x) \log Q(x)$$
ML/AI와의 연결: 크로스 엔트로피 손실함수

"놀라움"의 개념은 현대 딥러닝의 핵심 손실함수로 직결됩니다:

  • 크로스 엔트로피 손실(Cross-Entropy Loss): 분류 문제에서 가장 널리 쓰이는 손실함수. 모델의 예측 확률 분포($Q$)가 실제 분포($P$)와 얼마나 다른지를 측정합니다.
  • 모델이 정답을 높은 확률로 예측하면 → 놀라움 낮음 → 손실 작음
  • 모델이 정답을 낮은 확률로 예측하면 → 놀라움 높음 → 손실 큼
  • Binary Cross-Entropy: 이진 분류에서 $L = -[y\log\hat{y} + (1-y)\log(1-\hat{y})]$
  • KL-다이버전스: 두 확률분포의 차이를 측정. VAE(변분 오토인코더) 등에서 핵심적으로 사용됩니다.
